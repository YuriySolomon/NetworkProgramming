Введение в сетевое программирование.
Клиент-серверная технология

Сеть (network) - совокупность узлов и линий связи
Узел (node) - устройство, подключенное к сети (ПК. модем. принтер, ...)
	узел имеет сетевой адрес, например, IP
Связь - физический канал передачи данных между узлами.
Сетевая топология - способ организации связей между узлами
	"кольцо" - все узли образуют замкнутое соединение
	"шина" - "разомкнутое кольцо" - все узлы подключпются к одной шине
	"звезда" - в качестве узла может быть концентратор. подключающий
			   несколько других узлов и так в каждом из узлов

На одном узле (ПК) может одновременно работать несколько программ,
требующих передачи данных по сети. Для различения прграмм используется
понятие "сетевой порт" - номер, присваиваемый программе на узле.
Полный сетевой адрес программы (endpoint) - IP:port

Клиент-серверная технология: программный комплекс исполняется в виде
двух компонентов - клиента и сервера.
Сервер - постоянно активная составляющая, находящаяся в постоянном
	ожидании запросов (от клиентов)
Клиент - временно активная программа, посылающая серверу запросы и 
	получающая на них ответы.
Активных клиентов может быт несколько, в то время как сервер обычно один.

для того чтобы клиент мог подключиться к серверу. он должен обратиться
к той же endpoint, на которой работает сервер. Для передачи данных
также желательно "договориться" о кодировке. Набор подобных договоренностей
называют "протоколом"

Процесс обмена данными использует формализм "сокета". Сокет - это
	программный термин, скрывающий под собой весь процесс передачи данных,
	в операциях "записи в сокет" и "чтение из сокета".
	Открыть сокет - установить связь двух программ.
	(1) запись в сокет -----[техн.процесс]-------> (2) читает из сокета
